SM_CTRL Documentation

C++ S-function для управления генератором сигналов

Входные порты

  • START_IN - вход сигнала старта
  • D0..D7 - произвольные параметры типа double

Выходные порты

  • START_OUT - выход сигнала старта

По фронту START_IN происходит запись параметров D0..D7 в разделяемую память и формирование сигнала на сбор данных, после подтверждения начала сбора данных производится формирование фронта на выходе START_OUT

При работе с приложением simulink_a7dac назначение параметров следующее:

  • D0 - канал ЦАП 0 - частота сигнала
  • D1 - канал ЦАП 0 - начальная фаза сигнала
  • D2 - канал ЦАП 0 - амплитуда сигнала
  • D3 - канал ЦАП 1 - частота сигнала
  • D4 - канал ЦАП 1 - начальная фаза сигнала
  • D5 - канал ЦАП 1 - амплитуда сигнала
  • D6 - длина импульса в отсчётах тактовой частоты
  • D7 - резерв

При работе с другими приложениями назначение параметров может быть другим.

Основные функции и классы:

  • BaseBlock - базовый класс для создания S-Function из библиотеки easyLink
  • Block - класс S-Function sm_ctrl
  • Block::start() - начало сеанса моделирования
  • Block::outputs() - формирование выходных сигналов
  • TF_SimulinkShared - обмен с приложением через разделяемую память
  • TL_SharedMemoryWrapper - управление разделяемой памятью

Для реализации S-Function используется библиотека easyLink-3.3.0
